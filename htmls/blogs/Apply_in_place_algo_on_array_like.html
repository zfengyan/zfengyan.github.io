Apply in-place algorithms on array-like containers

During the development of an algorithm, we may encounter situations in which some changes towards certain data structures (e.g. an array or similar containers) need to happen in-place. Sometimes such approach would bring convenience to the developing process, however there are certain curcumstances that we need to pay attention to: 1. The developing algorithm is expected to be running in a multi-threaded environment. The hitch is very easy to be ignored: while you are altering the array, other threads may need to access the array as well and might not expect it to be modified during the process. 2. The array needs to be used later, or by another thread once the lock has been released. (About thread and the lock: Chernel youtube video). In this situation changing the content of the array on-the-fly could cause potential problems. Overall, if we are developing a new algorithm inside a large project, say, we are adding a new function to some code base. It is generally not a good idea to modify the content of the input as the input may be used later somewhere else inside the code base. But for own development, it might be fine. For example, sometimes we pass the vector by reference (more efficient) and inside the function we perform certain operations on that vector without the need to copy the entire vector.
